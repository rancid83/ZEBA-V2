# Skill: Codex 작업 흐름

## 목표
ZEBA 프로젝트에서 Codex가 탐색, 구현, 검증을 일관된 순서로 수행하게 한다.

## 기본 흐름
1. 요구사항을 짧게 재정의
2. `rg`로 관련 파일 탐색
3. 기존 패턴과 충돌 여부 확인
4. 필요한 경우 에이전트 분리
5. 수정 전 변경 범위 확정
6. 구현 후 타입 체크 또는 빌드 검증
7. 변경 파일과 잔여 리스크 정리

## 탐색 체크리스트
- 관련 페이지 엔트리 파일 확인
- 실제 렌더 컴포넌트 확인
- 타입/유틸/스토어 재사용 가능 여부 확인
- 스타일이 Tailwind인지 SCSS 모듈인지 확인

## 구현 체크리스트
- 불필요한 새 컴포넌트 생성 금지
- 기존 폴더 구조 유지
- 반응형과 빈 상태 UI 유지
- 런타임 비용 큰 애니메이션이나 이펙트 주의

## 검증 체크리스트
- `tsc --noEmit`
- 필요 시 `npm run build`
- 주요 사용자 플로우 회귀 여부

## 출력 원칙
- 결과보다 먼저 핵심 변경점 전달
- 파일 경로는 명확히 표시
- 못 한 검증은 이유를 명시
