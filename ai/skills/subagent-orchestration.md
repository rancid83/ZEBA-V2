# Skill: 서브에이전트 오케스트레이션

## 목표
Claude처럼 작업을 역할별 에이전트로 분리하되, ZEBA 코드베이스에서 충돌 없이 빠르게 진행한다.

## 언제 사용하나
- 기능 추가 범위가 2개 이상 모듈에 걸칠 때
- 탐색, 구현, 검증을 나누면 속도가 빨라질 때
- 같은 작업 안에 코드 수정과 리뷰가 모두 필요할 때

## 역할 분리 규칙

### 1. 메인 오케스트레이터
- 요구사항 정리
- 작업을 독립 단위로 분해
- 각 에이전트에 파일 책임 부여
- 최종 통합 및 우선순위 결정

### 2. Explorer
- 관련 파일 조사
- 기존 패턴 정리
- 구현 위치 제안

### 3. Worker
- 지정된 파일만 구현
- 파일 충돌 없는 범위 작업
- 구현 후 영향 범위 보고

### 4. Verifier
- 타입 체크, 빌드, 회귀 위험 검토
- 성능/UX 저하 여부 점검

## 파일 분배 원칙
- 한 파일은 한 시점에 한 에이전트만 주 담당
- 공용 타입 수정이 필요하면 먼저 Explorer가 영향 범위 정리
- 레이아웃과 비즈니스 로직은 가능하면 다른 에이전트에 분배
- 마지막 통합은 메인 에이전트가 수행

## 권장 작업 흐름
1. 요구사항을 한 줄로 요약
2. 관련 파일과 범위를 조사
3. 병렬 가능한 작업만 분리
4. 구현 에이전트에 파일 소유권을 명시
5. 검증 에이전트가 별도로 리뷰
6. 메인 에이전트가 결과 통합

## 프롬프트 템플릿
```md
Agent: codex-orchestrator
Skill: subagent-orchestration

목표:
[한 줄 목표]

제약:
- 수정 가능 파일: [파일 또는 폴더]
- 수정 금지 파일: [파일 또는 폴더]

원하는 분업:
- Explorer: 구조 분석
- Worker: 구현
- Verifier: 검증
```

## 금지사항
- 같은 파일을 여러 에이전트에 동시 할당
- 분석 없이 구현부터 시작
- 빌드 실패 상태로 통합
- 근거 없는 대규모 리팩토링
