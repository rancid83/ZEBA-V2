# Skill: 스타일 폴리시·반응형·모션 (ZEBA)

## 목표
레이아웃 조정 이후 **마감**: 반응형, 접근성, Framer Motion으로 인터랙션을 다듬는다.

## 반응형
- Tailwind 브레이크포인트(`sm:` `md:` `lg:` 등)로 주요 레이아웃 전환.
- 터치 타겟은 최소 ~44px 수준을 고려.
- 가로 스크롤이 생기면 원인(고정 width, overflow)을 줄인다.

## 접근성 (a11y)
- 인터랙티브 요소는 `button`/`a` 등 시맨틱 태그 사용.
- 아이콘만 있는 버튼에는 `aria-label`.
- 포커스가 보이도록; `outline-none` 남용 시 대체 포커스 스타일.
- 모달·드로어는 포커스 트랩·닫기 동작이 antd 기본에 맡겨져 있는지 확인.

## Framer Motion v12
- **필요한 곳만**: 페이지 진입, 섹션 reveal, 소량의 호버 피드백.
- `prefers-reduced-motion`을 고려할 수 있으면 고려 (과도한 필수 아님, 프로젝트 패턴 우선).
- 레이아웃 애니메이션은 성능(리플로우) 주의.

## 체크리스트
- [ ] 주요 뷰포트에서 스크롤·overflow 문제 없음
- [ ] 키보드만으로 주요 플로우 동작 가능
- [ ] 애니메이션이 콘텐츠를 가리지 않음

## 출력
1. 변경 파일 경로
2. a11y·반응형·모션 각각 한 줄 요약
3. 코드
