# Skill: 디자인 시스템·UI 스타일 (ZEBA)

## 목표
Ant Design, Tailwind v4, SASS 모듈을 **한 화면 안에서 역할이 겹치지 않게** 쓰고, 간격·색·타이포를 프로젝트 톤에 맞춘다.

## 역할 분담 가이드
| 용도 | 권장 |
|------|------|
| 레이아웃·그리드·간격·반응형 브레이크포인트 | Tailwind v4 유틸 |
| 복합 UI·폼·테이블·모달 | antd 컴포넌트 + 필요 시 `className` |
| 컴포넌트 전용 스코프·미세 조정 | `*.module.scss` |
| 전역 리셋·테마·antd 연동 | `globals.scss` |

## Tailwind v4
- 유틸 클래스 우선; arbitrary value는 `w-[200px]`, `text-[#hex]` 형식.
- `@apply`는 `globals.scss` / Tailwind 진입 CSS에서만 (기존 패턴 유지).

## Ant Design v6
- 이미 있는 `ConfigProvider`/테마가 있으면 그 범위 안에서 조정.
- 한 번에 많은 컴포넌트를 덮어쓰기보다, **해당 화면의 래퍼**에서 `className`·스타일로 제어.

## SASS 모듈
- 클래스명은 기존 BEM/모듈 패턴을 따른다.
- 다른 도메인 모듈 선택자에 깊게 의존하지 않는다.

## 체크리스트
- [ ] 모바일/태블릿/데스크톱에서 레이아웃이 깨지지 않음
- [ ] 호버/포커스/비활성 상태가 시각적으로 구분됨
- [ ] 색 대비가 읽기 가능한 수준
- [ ] 새 색·간격이 주변 컴포넌트와 이질적이지 않음

## 출력
1. 수정·추가 파일 경로
2. 스타일 변경 요약
3. 코드
